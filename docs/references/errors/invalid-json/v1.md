---
title: "Input file is not valid JSON"
description: "An input file passed to mif-cli or mif-mcp was not valid JSON."
---

**Type URI:** `https://modeled-information-format.github.io/mif-rs/references/errors/invalid-json/v1`
**HTTP status:** 400
**Exit code:** 2
**Emitted by:** `mif-cli` (`CliError::Json`), `mif-mcp` (`McpError::Json`)

## Message

```
failed to parse {path} as JSON: {source}
```

## Cause

`mif-cli validate`/`ingest`, or the `mif-mcp` `validate_mif_document`/
`ingest_mif_document` tools, read `path` successfully but it did not parse
as JSON — reached only for the already-JSON-LD input branch, since a
markdown-with-frontmatter input takes a different parse path entirely.

## Suggested fix

No suggested fix or code action is attached to this variant — the emitted
envelope carries only the standard RFC 9457 members plus `exit_code`.
