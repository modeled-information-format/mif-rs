---
title: "Failed to fetch from the ontology registry source"
description: "Fetching the registry index or a vendored ontology file from the resolved ontology source failed."
---

**Type URI:** `https://modeled-information-format.github.io/mif-rs/references/errors/registry-fetch-failed/v1`
**HTTP status:** 502
**Exit code:** 1
**Emitted by:** `mif-rh` (`MifRhError::RegistryFetch`)

## Message

```
failed to fetch from ontology source {registry_source}: {detail}
```

## Cause

`mif-rh` failed to fetch a file — either the registry index or a vendored
ontology — from the resolved ontology source, which is either a local
directory or an http(s) base URL. Note: the field carrying the source is
named `registry_source`, not `source`, because `thiserror` treats a field
literally named `source` as the error-chain cause and requires it to
implement `std::error::Error`, which a plain `String` does not.

## Suggested fix

**MaybeIncorrect** — check network access to the ontology source (or, for a
local directory source, that the path exists), then retry.
