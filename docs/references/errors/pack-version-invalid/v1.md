---
title: "Pack's declared version is not well-formed semver"
description: "A pack's declared version is not well-formed semver."
---

**Type URI:** `https://modeled-information-format.github.io/mif-rs/references/errors/pack-version-invalid/v1`
**HTTP status:** 422
**Exit code:** 2
**Emitted by:** `mif-rh` (`MifRhError::PackVersionInvalid`)

## Message

```
pack '{name}' {path} has no valid semver .version (got '{value}')
```

## Cause

This fires when a pack's declared version, read from its own manifest file,
does not parse as well-formed `X.Y.Z` semver. `mif-rh` needs a valid current
version before it can compute or apply a bump.

## Suggested fix

**MaybeIncorrect** — correct the version to well-formed X.Y.Z semver, then
retry.
