---
title: "Internal error extracting ontology metadata"
description: "An ontology definition passed schema validation but did not match the expected metadata shape."
---

**Type URI:** `https://modeled-information-format.github.io/mif-rs/references/errors/ontology-metadata-mismatch/v1`
**HTTP status:** 500
**Exit code:** 1
**Emitted by:** `mif-ontology` (`OntologyError::Deserialize`)

## Message

```
failed to extract ontology metadata from {path}: {source}
```

## Cause

The ontology definition at `path` passed schema validation, but could not be
deserialized into `OntologyMetadata`. This indicates a bug in `mif-ontology`
itself or a gap between `ontology.schema.json` and `OntologyMetadata` — not a
problem with the definition file.

## Suggested fix

**Unspecified** — this indicates a bug in `mif-ontology`. Report it
upstream.
