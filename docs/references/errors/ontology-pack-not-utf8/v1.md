---
title: "Fetched ontology pack is not valid UTF-8"
description: "A checksum-verified, freshly-fetched ontology pack's bytes were not valid UTF-8."
---

**Type URI:** `https://modeled-information-format.github.io/mif-rs/references/errors/ontology-pack-not-utf8/v1`
**HTTP status:** 422
**Exit code:** 1
**Emitted by:** `mif-rh` (`MifRhError::OntologyPackNotUtf8`)

## Message

```
ontology '{id}' file '{file}' is not valid UTF-8
```

## Cause

A checksum-verified, freshly-fetched ontology pack's bytes turned out not to
be valid UTF-8, so the file is rejected rather than lossily converted. A
lossy conversion would silently substitute replacement characters, which
would mean the diffed schema is no longer the exact schema that was already
hashed and fetched.

## Suggested fix

**Unspecified** — the registry-published ontology pack file is corrupt or was published with non-UTF-8 content; investigate upstream and re-publish a valid pack.
