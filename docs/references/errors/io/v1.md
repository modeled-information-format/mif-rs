---
title: "Failed to read an input file"
description: "A generic I/O failure reading a file, shared by mif-ontology, mif-cli, and mif-mcp."
---

**Type URI:** `https://modeled-information-format.github.io/mif-rs/references/errors/io/v1`
**HTTP status:** 500 by default, but **dynamically reclassified** — see below
**Exit code:** 1
**Emitted by:** `mif-ontology` (`OntologyError::Io`), `mif-cli` (`CliError::Io`), `mif-mcp` (`McpError::Io`)

## Message

```
failed to read {path}: {source}
```

## Cause

A file read failed at the OS level: an ontology definition file
(`mif-ontology`), or an input file passed to a `mif-cli` subcommand or
`mif-mcp` tool. All three crates share this exact slug and message shape
because the failure is the same conceptual problem regardless of which
crate's caller triggered it; the `instance` field (`urn:{crate}:io`)
identifies which crate actually emitted a given occurrence.

**Status is not fixed.** `mif_problem::classify_io_error` inspects the
underlying `std::io::ErrorKind` and overrides the status at emission time:

| `ErrorKind` | Status | Applicability |
|---|---|---|
| `NotFound` | 404 | MaybeIncorrect — verify the path exists |
| `PermissionDenied` | 403 | MaybeIncorrect — verify permissions |
| anything else | 500 | Unspecified — genuine I/O fault, not a path mistake |

## Suggested fix

Depends on the classified status (see table above) — always **MaybeIncorrect**
for 404/403, **Unspecified** for a genuine I/O fault at 500.
