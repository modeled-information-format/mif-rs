---
title: "Ontology pack YAML failed to parse"
description: "An ontology pack YAML file failed to parse"
---

**Type URI:** `https://modeled-information-format.github.io/mif-rs/references/errors/ontology-pack-invalid-yaml/v1`
**HTTP status:** 422
**Exit code:** 4
**Emitted by:** `mif-rh` (`MifRhError::OntologyPackYaml`)

## Message

```
failed to parse ontology pack {path} as YAML: {source}
```

## Cause

This is the direct equivalent of `yq` failing to read an ontology's
`extends`/`entity_types`/full YAML — a fail-closed abort that matches rht's
own bash exit code 4. It fires when an ontology pack YAML file is read
successfully but its contents fail to parse, so the pack cannot be loaded
into the resolution engine at all.

## Suggested fix

**MaybeIncorrect** — fix the YAML syntax error in the ontology pack, then
retry.
