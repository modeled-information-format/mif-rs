---
title: "Failed to read an ontology definition file"
description: "A generic I/O failure reading an ontology definition file."
---

**Type URI:** `https://modeled-information-format.github.io/mif-rs/references/errors/mif-ontology-io/v1`
**HTTP status:** 500 by default, but **dynamically reclassified** — see below
**Exit code:** 1
**Emitted by:** `mif-ontology` (`OntologyError::Io`)

## Message

```
failed to read {path}: {source}
```

## Cause

A file read failed at the OS level while loading an ontology definition
file from a corpus directory (`mif_ontology::load_corpus_from_dir`).
`mif-cli` and `mif-mcp` each report the identical conceptual failure
through their own `mif-cli-io`/`mif-mcp-io` problem types instead of this
one — each crate's `Io` variant maps to its own distinct, versioned slug,
not a slug shared across crates.

**Status is not fixed.** `mif_problem::classify_io_error` inspects the
underlying `std::io::ErrorKind` and overrides the status at emission time:

| `ErrorKind` | Status | Applicability |
|---|---|---|
| `NotFound` | 404 | MaybeIncorrect — verify the path exists |
| `PermissionDenied` | 403 | MaybeIncorrect — verify permissions |
| anything else | 500 | Unspecified — genuine I/O fault, not a path mistake |

## Suggested fix

Depends on the classified status (see table above) — always **MaybeIncorrect**
for 404/403, **Unspecified** for a genuine I/O fault at 500.
