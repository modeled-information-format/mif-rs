---
title: "Ontology catalog file does not exist"
description: "The catalog file is missing"
---

**Type URI:** `https://modeled-information-format.github.io/mif-rs/references/errors/catalog-missing/v1`
**HTTP status:** 404
**Exit code:** 3
**Emitted by:** `mif-rh` (`MifRhError::CatalogMissing`)

## Message

```
catalog file {path} does not exist
```

## Cause

This fires when the catalog file (`.claude/enabled-packs.json`) is missing
at the path a resolution or review operation expects it at. Without this
file, `mif-rh` has no record of which ontology packs are enabled, so it
cannot proceed.

## Suggested fix

**MachineApplicable** — run rht's scripts/sync-packs.sh (or mif-rh-cli's
equivalent) to generate the catalog, then retry.
