title: "CHANGELOG has no anchor to insert the new version under"
description: "CHANGELOG.md has neither an ## [Unreleased] anchor to insert a new section under, nor an existing section for the new version."
---

**Type URI:** `https://modeled-information-format.github.io/mif-rs/references/errors/changelog-anchor-missing/v1`
**HTTP status:** 422
**Exit code:** 2
**Emitted by:** `mif-rh` (`MifRhError::ChangelogAnchorMissing`)

## Message

```
{path} has no '## [Unreleased]' anchor to insert the new version under (nor an existing section for it)
```

## Cause

This fires when `CHANGELOG.md` has neither an `## [Unreleased]` heading for
`mif-rh` to insert the new version's section under, nor an existing section
already matching the new version. Without one of those two anchors, it has
no safe place to write the entry.

## Suggested fix

**MaybeIncorrect** — add an '## [Unreleased]' section to the CHANGELOG,
then retry.
