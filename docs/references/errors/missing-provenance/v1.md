---
title: "Corpus import contains findings with no provenance block"
description: "One or more findings in a corpus import lack a provenance block, so the import is aborted."
---

**Type URI:** `https://modeled-information-format.github.io/mif-rs/references/errors/missing-provenance/v1`
**HTTP status:** 422
**Exit code:** 1
**Emitted by:** `mif-rh` (`MifRhError::MissingProvenance`)

## Message

```
{count} finding(s) lack a provenance block; import aborted (provenance must be preserved)
```

## Cause

This fires when one or more findings in a corpus import lack a provenance
block. Per SPEC §8a, provenance must survive an import, so the import is
aborted rather than silently dropping it. The variant carries a `count`
field (how many findings lack provenance) alongside a `paths` field listing
the paths of the findings missing provenance.

## Suggested fix

**MaybeIncorrect** — add a provenance block to every listed finding before retrying the import.
