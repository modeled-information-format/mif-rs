---
title: "Version is not well-formed semver"
description: "A version string is not well-formed X.Y.Z semver."
---

**Type URI:** `https://modeled-information-format.github.io/mif-rs/references/errors/version-not-semver/v1`
**HTTP status:** 422
**Exit code:** 2
**Emitted by:** `mif-rh` (`MifRhError::VersionNotSemver`)

## Message

```
version is not well-formed semver: {value}
```

## Cause

This fires when a version string `mif-rh` needs to parse is not well-formed
`X.Y.Z` semver. It can surface anywhere a version is read back off disk or
supplied as an argument and doesn't match the expected three-part numeric
shape.

## Suggested fix

**MaybeIncorrect** — correct the version to well-formed X.Y.Z semver, then
retry.
