---
title: "Post-write self-verification found an unexpected file"
description: "A post-write self-verification found a file that did not update to the expected new value."
---

**Type URI:** `https://modeled-information-format.github.io/mif-rs/references/errors/verification-failed/v1`
**HTTP status:** 500
**Exit code:** 2
**Emitted by:** `mif-rh` (`MifRhError::VerificationFailed`)

## Message

```
verification failed: {path} was not updated as expected
```

## Cause

This fires when `mif-rh`'s post-write self-verification step re-reads a file
it just wrote and finds it does not carry the expected new value. It signals
that a write silently failed to take effect, or applied only partially.

## Suggested fix

**Unspecified** — inspect the named file directly — the write may have
partially applied. This indicates a bug in the bump logic, not a
caller-side fix.
