---
title: "Topic's ontology map carries no typed entities to draft from"
description: "A topic's ontology-map.json carries no typed entity types to mine an ontology draft from."
---

**Type URI:** `https://modeled-information-format.github.io/mif-rs/references/errors/no-entity-types-found/v1`
**HTTP status:** 422
**Exit code:** 1
**Emitted by:** `mif-rh` (`MifRhError::NoEntityTypesFound`)

## Message

```
no entity types found in reports/{topic}/ontology-map.json — nothing to draft an ontology from
```

## Cause

A topic's `ontology-map.json` carries no typed entity types, so there is
nothing for the ontology-authoring flow to mine a draft from. This is a
data-availability check, not a structural one — the file exists and parses,
it simply has no typed findings yet.

## Suggested fix

**MaybeIncorrect** — run /ontology-review on the topic first so its findings get typed, then retry.
