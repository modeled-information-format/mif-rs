---
title: "Pack component name is ambiguous"
description: "A --pack component name resolves to more than one directory."
---

**Type URI:** `https://modeled-information-format.github.io/mif-rs/references/errors/pack-ambiguous/v1`
**HTTP status:** 422
**Exit code:** 2
**Emitted by:** `mif-rh` (`MifRhError::PackAmbiguous`)

## Message

```
pack '{name}' is ambiguous: more than one packs/<family>/{name} directory exists
```

## Cause

This fires when a `--pack` component name matches more than one
`packs/<family>/<name>` directory across different families, so `mif-rh`
cannot tell which one the caller means.

## Suggested fix

**MaybeIncorrect** — check the component name against
packs/<family>/<name>/ and retry.
