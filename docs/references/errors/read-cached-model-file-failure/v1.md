title: "Failed to read a locally cached model file"
description: "Failed to read a locally cached embedding model file."
---

**Type URI:** `https://modeled-information-format.github.io/mif-rs/references/errors/read-cached-model-file-failure/v1`
**HTTP status:** 500 by default, but **dynamically reclassified** — see below
**Exit code:** 1
**Emitted by:** `mif-embed` (`EmbedError::ReadFile`)

## Message

```
failed to read '{path}': {source}
```

## Cause

Reading a model file already fetched and cached locally (`config.json`,
`tokenizer.json`, or `model.safetensors`) failed at the OS level.

**Status is not fixed.** `mif_problem::classify_io_error` inspects the
underlying `std::io::ErrorKind` and overrides the status at emission time —
see the shared [`io/v1`](/mif-rs/references/errors/io/v1/) reference for the
full reclassification table (404 `NotFound`, 403 `PermissionDenied`, 500
otherwise).

## Suggested fix

Depends on the classified status — **MaybeIncorrect** for 404/403,
**Unspecified** for a genuine I/O fault at 500.
