---
title: "Ref-target schema has no $id"
description: "A $ref dependency schema has no $id, so it cannot be registered as a $ref target."
---

**Type URI:** `https://modeled-information-format.github.io/mif-rs/references/errors/ref-schema-missing-id/v1`
**HTTP status:** 422
**Exit code:** 2
**Emitted by:** `mif-rh` (`MifRhError::RefSchemaMissingId`)

## Message

```
schema {path} has no $id — it cannot be registered as a $ref target
```

## Cause

This fires when a `$ref` dependency schema has no `$id`. Without a `$id`,
there is nothing for the main schema to resolve its reference against, so
the dependency schema cannot be registered as a valid `$ref` target.

## Suggested fix

**MaybeIncorrect** — add a $id to the dependency schema, then retry.
