---
title: "Concordance is not a valid graph"
description: "A concordance file parsed but is not a valid graph — it has no .nodes array to project the corpus atlas from."
---

**Type URI:** `https://modeled-information-format.github.io/mif-rs/references/errors/invalid-concordance/v1`
**HTTP status:** 422
**Exit code:** 2
**Emitted by:** `mif-rh` (`MifRhError::InvalidConcordance`)

## Message

```
concordance is not a valid graph (no .nodes array): {path}
```

## Cause

This fires when a concordance file parses successfully but is not a valid
graph — it has no `.nodes` array. The corpus atlas is a projection of the
spine, and without a valid graph there is nothing to project it from.

## Suggested fix

**MaybeIncorrect** — build the concordance first (scripts/build-concordance.sh) before synthesizing the corpus atlas.
