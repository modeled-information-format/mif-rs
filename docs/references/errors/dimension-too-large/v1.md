---
title: "Vector has too many dimensions to store"
description: "The vector to store has more components than the embeddings table's dim column can represent."
---

**Type URI:** `https://modeled-information-format.github.io/mif-rs/references/errors/dimension-too-large/v1`
**HTTP status:** 400
**Exit code:** 2
**Emitted by:** `mif-store` (`StoreError::DimensionTooLarge`)

## Message

```
vector has {len} dimensions, which cannot be represented in the embeddings table
```

## Cause

`VectorStore::upsert`'s vector length could not be represented as an `i64`
for the `dim` column — practically unreachable with real embedding models
(`mif-embed` always produces 384-dimensional vectors), but defended against
directly.

## Suggested fix

**Unspecified** — supply a shorter embedding vector.
