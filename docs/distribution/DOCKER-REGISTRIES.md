# mif-rs Container Images â€” reference

One container image per binary crate, built and pushed by
`.github/workflows/release-docker.yml`, invoked from
`.github/workflows/pipeline.yml`'s `docker` job.

## Registry

| Field | Value |
| --- | --- |
| Registry | GitHub Container Registry (`ghcr.io`) â€” the only registry this repo publishes to |
| Image (mif-cli) | `ghcr.io/modeled-information-format/mif-rs/mif-cli` |
| Image (mif-mcp) | `ghcr.io/modeled-information-format/mif-rs/mif-mcp` |
| Platforms | `linux/amd64`, `linux/arm64` |
| Base image | `gcr.io/distroless/cc-debian12` (pinned by digest) |

## Tags

Generated by `docker/metadata-action`, identical scheme for both images:

| Tag pattern | Applies when |
| --- | --- |
| `<branch>` | Push to a branch |
| `pr-<number>` | Pull request |
| `<major>.<minor>.<patch>` | Semver tag (`v*.*.*`) |
| `<major>.<minor>` | Semver tag |
| `<major>` | Semver tag |
| `sha-<short-sha>` | Every build |
| `latest` | Push to the default branch |

## Pull and run

```bash
docker pull ghcr.io/modeled-information-format/mif-rs/mif-cli:latest
docker run --rm ghcr.io/modeled-information-format/mif-rs/mif-cli:latest validate document.json

docker pull ghcr.io/modeled-information-format/mif-rs/mif-mcp:latest
docker run --rm ghcr.io/modeled-information-format/mif-rs/mif-mcp:latest
```

## Build gating

| Condition | Behavior |
| --- | --- |
| `has-bin-target` output (`pipeline.yml`'s `gate` job) is `false` | `docker` job does not run |
| Pull request | Image is built, not pushed (`push: false`) |
| Push to a branch or tag | Image is built and pushed |

`has-bin-target` is derived from `cargo metadata` dynamically â€” true from
day one here (`mif-cli`, `mif-mcp` are real workspace members), and stays
correct if a third binary crate joins later without a workflow change.

## Signing and verification

Every pushed image is signed and attested by the org's centralized signer,
then fail-closed verified before the pipeline continues:

| Job | Workflow called | Purpose |
| --- | --- | --- |
| `docker-sign` | `modeled-information-format/.github`'s `sign-and-attest.yml` | Signs the image, attaches SLSA provenance |
| `docker-verify` | `modeled-information-format/.github`'s `verify-attestation.yml` | Fail-closed verification of the signature/provenance before the run is considered successful |

Both jobs run once per binary image (matrixed over `mif-cli`/`mif-mcp`),
each looking up its own image's digest â€” the digests are collected from
`release-docker.yml`'s per-bin build matrix via artifact upload/download,
since a matrix job's `outputs` in GitHub Actions only exposes its last
completed cell's value.

## Manual verification

```bash
gh attestation verify \
  --owner modeled-information-format \
  oci://ghcr.io/modeled-information-format/mif-rs/mif-cli:latest
```

behind this signing chain.
